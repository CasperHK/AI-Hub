# 排名與基準測試功能 - 使用示例與截圖說明

## 📱 使用示例

### 場景 1：選擇圖像生成工具
**用戶需求：** "我想找一個最好的 AI 圖像生成工具，主要用於藝術創作"

**操作步驟：**
1. 在主頁點擊「圖像生成」(🎨) 分類
2. 向下滾動，點擊「查看此分類的排名與基準測試」按鈕
3. 進入圖像生成基準測試頁面
4. 點擊使用案例篩選中的「🎨 藝術創作」
5. 查看排名：
   - #1: Midjourney (8.4/10) - **最適合藝術創作**
   - #2: DALL-E 3 (8.2/10)
   - #3: Stable Diffusion (8.0/10)
6. 點擊 Midjourney 卡片查看詳細信息
7. 對比優缺點，做出決定

**結果：** 用戶選擇 Midjourney，因為它在「圖像品質」和「風格多樣性」上評分最高

### 場景 2：比較程式碼助手
**用戶需求：** "我需要一個適合網頁開發的程式碼助手，需要 IDE 集成支援"

**操作步驟：**
1. 在主頁點擊「程式碼助手」(💻) 分類
2. 點擊「查看此分類的排名與基準測試」按鈕
3. 進入程式碼助手基準測試頁面
4. 在「使用案例篩選」中選擇「🌐 網頁開發」
5. 使用排序保持「按總體評分排序」
6. 查看排名結果：
   - #1: Cursor (8.4/10) - IDE 整合 9/10
   - #2: Tabnine (8.2/10) - IDE 整合 9/10
   - #3: GitHub Copilot (8.0/10) - IDE 整合 9/10
7. 使用快速對比表格：
   ```
   工具名         總體評分   程式碼品質   IDE 整合   上下文理解
   Cursor         8.4        9/10        8/10      9/10
   Tabnine        8.2        7/10        9/10      7/10
   GitHub Copilot 8.0        8/10        9/10      8/10
   ```
8. 選擇 Cursor (最好的上下文理解)

**結果：** 用戶選擇 Cursor，因為它具有最佳的上下文理解能力

### 場景 3：分析 AI 智能體工具
**用戶需求：** "我要建立一個完整的工作流程自動化系統，需要支援多個智能體協調"

**操作步驟：**
1. 在主頁點擊「Agentic AI」(🤖) 分類
2. 點擊「查看此分類的排名與基準測試」按鈕
3. 進入 Agentic AI 基準測試頁面
4. 查看「效能基準測試指標」：
   - 平均回應時間 (s)
   - API 呼叫速度 (ms)
   - 並行處理能力 (任務數)
   - 成功率 (%)
5. 檢查「功能評分標準」：
   - 工作流程自動化能力 (25% 權重) ⭐
   - API 與擴充性 (25% 權重) ⭐
   - 易用性 (20% 權重)
   - AI 模型選擇 (15% 權重)
   - 社群與支援 (15% 權重)
6. 查看工具排名卡片：
   ```
   #1 Claude (8.6/10)
   ✓ 優點：強大推理、豐富API、MCP伺服器、企業安全
   ✗ 缺點：價格高、等待時間長
   
   #2 CrewAI (8.2/10)
   ✓ 優點：開源、支援多模型、企業級、文件豐富
   ✗ 缺點：需要編程、部署複雜
   
   #3 Gumloop (7.8/10)
   ✓ 優點：視覺化建構、易上手、多智能體、快速部署
   ✗ 缺點：模型有限、進階功能陡峭
   ```
7. 在使用案例篩選中選擇「⚙️ 工作流程自動化」
8. 基於比較，選擇 Claude (推理最強) 或 CrewAI (開源高度自訂)

**結果：** 用戶根據需求選擇：
- 快速上線？選 Claude
- 完全自訂？選 CrewAI
- 預算有限？選 Gumloop

## 📊 數據顯示邏輯

### 排名計算
```
總體評分 = Σ(工具在各維度評分 × 對應權重) / 100

例子：Midjourney 圖像生成
= (圖像品質 10 × 30% + 風格多樣性 9 × 20% + 提示詞理解 9 × 20% 
   + 編輯功能 7 × 15% + 價格效能比 7 × 15%)
= (3.0 + 1.8 + 1.8 + 1.05 + 1.05) / 10
= 8.4 / 10
```

### 排序邏輯
- **按評分排序**：從高分到低分 (8.4 → 8.2 → 8.0)
- **按名稱排序**：按字母順序 (DALL-E → Midjourney → Stable)

### 篩選邏輯
- **使用案例篩選**：顯示「最適為」包含選定案例的工具
- 多選時：顯示符合任何一個案例的工具

## 🎯 推薦使用流程

### 第一次使用
1. **瀏覽所有指標** - 理解評分維度和基準指標
2. **查看完整排名** - 了解該分類的整體情況
3. **詳細閱讀首選工具** - 深入了解 #1 工具的所有信息
4. **快速對比前 3** - 比較排名前 3 的工具差異

### 有明確需求時
1. **選擇使用案例** - 篩選最相關的工具
2. **查看詳細卡片** - 閱讀優缺點分析
3. **查看快速表格** - 橫向對比關鍵指標
4. **查看工具詳情頁** - 點擊工具卡片查看完整信息

### 對比多個工具
1. **按「總體評分」排序**
2. **查看快速對比表格** - 快速掃描數據
3. **點擊感興趣的工具** - 查看詳細信息
4. **記錄優缺點差異** - 幫助決策

## 💡 視覺設計亮點

1. **色碼系統**
   - 紅色：優秀 (9.0+)
   - 橙色：很好 (8.0-8.9)
   - 黃色：良好 (7.0-7.9)
   - 藍色：及格 (6.0-6.9)

2. **圖標使用**
   - 分類圖標 (🎨, 💻, 🎤 等)
   - 符號標記 (✓ 優點, ✗ 缺點)
   - 指標圖標 (📊, ⭐, 🔧 等)

3. **排版層級**
   - H1：分類名稱
   - H2：主要區段標題
   - H3-H4：子標題
   - 高對比度：重要數據

4. **互動反饋**
   - 按鈕懸停效果
   - 篩選按鈕高亮
   - 卡片陰影效果
   - 進度條動畫
